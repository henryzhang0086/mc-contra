# MC 魂斗罗 — 5A 级生产计划 (Production Plan)

> 目标：把现有可玩原型升级为**工业级、专业游戏公司 5A 出品标准**的作品。
> 方向（已确认）：**混合美术/音频**（程序化为底 + 关键真实资源，构建时 base64 内联，保持单文件可交付）·
> **垂直切片优先** · **移动+桌面并重** · 加入**本地双人合作** + **元进程/解锁**。

---

## 0. 设计支柱 (Design Pillars)

一切取舍服从这 5 根支柱；任何功能若不服务于某根支柱，砍掉或延后。

1. **爽快跑射手感** — 跟手、即时、有重量；开火/命中/受击/跳跃都"打到肉里"。
2. **高辨识度演出** — Boss 与场景演出要"大"，但子弹与威胁**永远读得清**（敌我弹幕配色/形状强对比）。
3. **双人共斗的混乱快乐** — 同屏 2P 的协作与搞笑时刻是核心卖点。
4. **处处可玩** — 手机与桌面都一流；无障碍、可重映射、手柄、触控并列一等公民。
5. **工业级稳健** — 锁定 60fps、零崩溃、自动化测试守门、可一键发布。

## 1. 质量基线 (Definition of Done — "5A 标杆"验收标准)

垂直切片必须**全部满足**才算达标；这套标准随后复制到全部内容。

### 性能 / 技术
- [ ] 逻辑锁定 60Hz，**渲染插值**后在 120/144Hz 完全丝滑（当前缺插值）。
- [ ] 帧时间预算 < 8ms（留 1 倍余量）；**对象池**消除子弹/粒子 GC 抖动。
- [ ] 输入延迟 ≤ 2 帧；所有关键动作有**缓冲/宽容**（coyote、跳跃缓冲、开火缓冲、瞄准吸附）。
- [ ] 启动到可玩 < 2s；资源加载有进度条与失败兜底。

### 手感 / 反馈 (Game Feel)
- [ ] 每个动作 **3 层反馈**：视觉（粒子/闪光/顿帧/相机震/拖尾）+ 音频 + 触觉（震动/手柄）。
- [ ] 武器有**后坐力 + 枪口闪光 + 弹道拖尾 + 命中迸溅**，各武器手感差异鲜明。
- [ ] 相机：朝向前瞻 + 平滑 + 受击/爆炸 **trauma 震动模型** + Boss 入场运镜。

### 美术 / 演出
- [ ] 统一**色板**与**像素字体**；UI 一致的设计语言。
- [ ] **多层视差**背景 + 氛围粒子 + 动态光照/暗角；昼夜或环境演出。
- [ ] 角色/敌人有**多帧动画**（待机/跑/跳/受击/死亡）与朝向；Boss 分阶段换演出。
- [ ] 至少 1 处**场景演出 (set-piece)**（如崩塌、追逐、背景大事件）。

### 音频
- [ ] **自适应分层配乐**：探索/战斗/Boss 按强度动态混音（vertical remixing）。
- [ ] 空间声像（左右声像随位置）、爆炸低频、命中 ducking；≥ 20 个差异化音效。

### UX / 无障碍 / 平台
- [ ] 完整**新手引导**（首关前 30 秒教会跑/跳/瞄/射/换武器）。
- [ ] **可重映射按键** + **手柄支持**（Gamepad API + 震动）+ 触控三套并列一流。
- [ ] 无障碍：色盲模式、减动效、可调难度、握持/切换开火、字号、屏幕安全区。
- [ ] 暂停即时无延迟；菜单可键鼠/触控/手柄全操作。

### 内容 / 关卡
- [ ] 该关有**节奏曲线**：铺垫 → 压力 → 喘息 → 高潮 → Boss；≥3 种敌人编排组合。
- [ ] Boss ≥ 3 阶段、≥ 4 套招式、有"破绽窗口"教学引导。

### 质量保证 / 发布
- [ ] 自动**冒烟 + 视觉回归 + 性能预算**测试全过；崩溃零（含错误上报）。
- [ ] **存档版本化**（迁移逻辑）；i18n **中/英双语**字符串表。
- [ ] 一键构建单文件 + 一键发布 Pages；附宣传图/截图/GIF。

## 2. 里程碑路线图 (Milestones)

垂直切片优先：**先用 M0+M1 立标杆**，再用 M2 系统化、M3 横向铺开、M4 打磨发布。
（工作量为相对规模 S/M/L/XL，非日历）

### M0 · 制作管线与技术地基 (Foundation) — L
先把"做 5A 的工具"造出来，否则后面每一步都翻倍痛苦。
- **资源管线**：`build.mjs` 升级为可把 PNG 精灵表 / 音频 base64 内联进单文件；资源清单 + 异步加载 + 进度条。
- **数据驱动**：把关卡/敌人/武器/Boss 招式抽成 JSON 式数据表（现在是硬编码），便于快速调参与扩展。
- **渲染插值**：固定步长 + 渲染插帧（存上一帧状态，按 alpha 插值绘制）→ 高刷丝滑。
- **对象池**：子弹/粒子/飘字/敌弹池化，消除 GC。
- **调试工具升级**：实时调参面板（`?debug`）、帧时间/实体数 HUD、关卡快速跳转、无敌/给武器、时间缩放。
- **测试地基**：扩展 playwright —— 视觉回归（基线截图比对）、性能采样（帧时间）、输入回放。
- **i18n 框架**：字符串表 + `t(key)`，中/英切换。
- ✅ 验收：插值开关可见丝滑差异；单文件能内联并加载一张精灵 + 一段音乐；调试面板可改数值即时生效。

### M1 · 垂直切片 (Vertical Slice — 5A 标杆关) — XL ⭐ 核心
把**第一关 + 要塞核心 Boss** 打磨到完全满足第 1 节全部验收标准。这是整个项目的"质量宪法"。
- **手感大改**：后坐力/枪口闪/拖尾/命中迸溅/顿帧/trauma 相机/瞄准吸附；5 把武器手感拉开差距。
- **美术升级**：像素字体 + 统一色板；士兵与 3 种敌人多帧动画；多层视差 + 氛围光效；Boss 分阶段演出。
- **音频升级**：该关自适应分层 BGM（探索/战斗/Boss）+ 20+ 音效 + 声像 + ducking。
- **关卡设计**：重做第一关为有节奏曲线的"教学+展示"关，含 1 处场景演出。
- **新手引导**：前 30 秒情境化教学（非弹窗，靠关卡设计引导）。
- **平台一流**：手柄支持 + 可重映射 + 触控手感复检（横屏/安全区/连发）。
- ✅ 验收：第 1 节清单**全绿**；找 3 个真人盲测，单手机/单桌面通关无卡顿无困惑。

### M2 · 系统化 (Systems) — L
把切片确立的标准变成贯穿全局的系统。
- **本地双人合作 (2P)**：同屏双角色（P1 键盘左区/P2 键盘右区 + 双手柄）、独立命数/复活、共享镜头与"濒死救援"、UI 双血条。
- **元进程/解锁**：通关/积分解锁角色皮肤、起始武器、难度模式；解锁画廊；存档版本化迁移。
- **完整游戏流程**：标题 → 模式选择（单人/双人/难度）→ 关卡 → 结算 → 解锁 → 标题；继续/重试/连续币（continue）。
- **无障碍全套**：色盲/减动效/难度/字号/握持射击/重映射，统一进设置。
- ✅ 验收：双人可同屏通关切片关；解锁链路打通并存档；无障碍项全部生效。

### M3 · 内容铺开 (Content Scale-Up) — XL
用 M1 的标准与 M0 的管线，批量产出达标内容。
- 关卡：扩到 **4–6 关**，每关独特生物群系、机关、节奏与 set-piece。
- Boss：每关 1 个，**多阶段多招式**；新增小 Boss/精英怪。
- 敌人：扩到 **8–10 种**，行为差异化、可组合编排。
- 武器/道具：补齐至 6–8 种 + 武器升级（同枪叠加）。
- 模式：Boss Rush、无尽生存、时间挑战（接元进程/排行）。
- ✅ 验收：每关都过第 1 节清单；难度曲线整体平滑。

### M4 · 打磨与发布 (Polish & Ship) — L
- 全局**手感/演出抛光周**（juice pass）、音乐混音、UI 动效统一。
- 性能：低端机/手机实测达标，电量与发热优化，长会话内存稳定。
- QA：全自动化测试 + 真人测试矩阵（机型/浏览器/输入方式）。
- 发布：宣传截图/GIF/预告、商店式落地页、SEO、（可选）排行榜后端、隐私说明、analytics 与崩溃上报。
- ✅ 验收：发布候选版（RC）通过全部门禁；可对外正式发布。

## 4. 现状差距清单 (从当前 v1.0.0 出发要补的硬骨头)

- 无渲染插值（高刷可更丝滑）· 无对象池（GC 抖动隐患）
- 无手柄支持 · 无可重映射 · 无新手引导 · 无难度/无障碍选项
- 音频偏简单（需真实分层配乐 + 声像 + ducking + 更大音效库）
- 敌人/角色为矢量无多帧动画（需精灵动画）· 演出与 set-piece 不足
- 无双人 · 无元进程/解锁 · 存档未版本化 · 无 i18n
- 测试仅冒烟（需视觉回归 + 性能预算 + 真人测试矩阵）

## 5. 建议的第一步

> 推荐从 **M0（地基）+ M1（垂直切片）** 启动——先把工具与标杆立住，质量基线一旦确立，后续全是"复制粘贴式"高效产出。

可选的启动方式：
- **A. 全量推进 M0**：一次性把资源管线/插值/对象池/调试面板/i18n/测试地基都搭好（推荐，地基决定上限）。
- **B. 先做最有"体感"的一项**：例如先上**渲染插值 + 手感抛光（后坐力/拖尾/顿帧/trauma 相机）**，立刻能感到"变高级"。
- **C. 先出双人**：如果你最想先玩到 2P，可优先做本地双人，再回头补地基。

确认从哪个启动，我就开工（每个里程碑我会按这套验收标准自测 + 截图 + 真机/headless 验证后再交付）。
